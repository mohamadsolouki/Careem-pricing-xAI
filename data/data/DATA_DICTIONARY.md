# Data Dictionary — Dubai Ride-Hailing Mirror Dataset 2025
## XPrice Project | MIT 622 Final Project | Group 1

Generated by `data/generate_dataset.py`

- Records: 165,000
- Features: 62 columns
- Period: Jan 1, 2025 to Dec 31, 2025
- Geography: Dubai plus Sharjah cross-border trips
- Primary target: `final_price_aed`

## Identifiers
| Column | Type | Description |
|--------|------|-------------|
| ride_id | str | Synthetic unique ride identifier in `RID...` format |
| customer_id | str | Synthetic anonymized customer identifier |
| captain_id | str | Synthetic anonymized captain identifier |

## Temporal Features
| Column | Type | Description |
|--------|------|-------------|
| timestamp | datetime | Full booking timestamp |
| date | date | Booking date |
| hour | int | Hour of booking, 0 to 23 |
| minute | int | Minute of booking, 0 to 59 |
| day_of_week | int | Day index where Monday = 0 and Sunday = 6 |
| day_name | str | Day name derived from timestamp |
| week_of_year | int | ISO week number |
| month | int | Month number, 1 to 12 |
| month_name | str | Month name |
| quarter | int | Calendar quarter, 1 to 4 |
| is_weekend | bool | `True` for UAE weekend days used in this dataset, Friday or Saturday |
| is_peak_hour | bool | `True` for RTA peak pricing windows |
| is_late_night | bool | `True` from 22:00 to 05:59 |
| is_offpeak | bool | `True` when ride is neither peak nor late night |
| is_ramadan | bool | `True` if the ride falls within Ramadan 2025 |
| is_uae_public_holiday | bool | `True` if the ride date is a UAE public holiday in the calendar used by the generator |
| is_suhoor_window | bool | `True` for Ramadan pre-dawn demand window |
| is_iftar_window | bool | `True` for Ramadan sunset demand spike window |

## Event Features
| Column | Type | Description |
|--------|------|-------------|
| active_event | str | Active named event, or `None` |
| event_type | str | Event category such as shopping festival, sports event, trade show, or holiday |
| event_demand_multiplier | float | Event-driven uplift applied to demand |

## Weather Features
| Column | Type | Description |
|--------|------|-------------|
| temperature_c | float | Simulated ambient temperature in Celsius |
| humidity_pct | float | Simulated humidity percentage |
| is_rain | bool | `True` when rain occurs |
| is_sandstorm | bool | `True` when sandstorm conditions occur |
| weather_demand_factor | float | Weather-driven uplift applied to demand |

## Geographic Features
| Column | Type | Description |
|--------|------|-------------|
| pickup_zone | str | Pickup zone name |
| dropoff_zone | str | Dropoff zone name |
| pickup_lat | float | Pickup latitude jittered around zone centroid |
| pickup_lon | float | Pickup longitude jittered around zone centroid |
| dropoff_lat | float | Dropoff latitude jittered around zone centroid |
| dropoff_lon | float | Dropoff longitude jittered around zone centroid |
| pickup_area_type | str | Pickup zone demand tier label: `High`, `Medium`, or `Low` |
| dropoff_area_type | str | Dropoff zone demand tier label: `High`, `Medium`, or `Low` |
| is_airport_ride | bool | `True` if either endpoint is DXB Airport |
| is_intrazone_trip | bool | `True` if pickup and dropoff are in the same zone |

## Trip Features
| Column | Type | Description |
|--------|------|-------------|
| route_distance_km | float | Simulated routed trip distance in kilometers |
| salik_gates | int | Number of Salik toll gates crossed |
| salik_cost_aed | float | Salik toll cost in AED |
| product_type | str | Product booked, for example `Comfort`, `Executive`, `Hala Taxi`, `Premier`, or `MAX` |
| is_hala_product | bool | `True` for regulated taxi-style Hala products |
| payment_method | str | Payment method used |
| is_careem_plus | bool | `True` when payment method is `Careem Plus` |
| trip_duration_min | float | Simulated trip duration in minutes |
| avg_speed_kmh | float | Simulated average trip speed |

## Demand And Supply Features
| Column | Type | Description |
|--------|------|-------------|
| demand_index | float | Composite demand score combining zone, events, weather, time-of-day, weekend effects, and Ramadan windows |
| captain_availability_score | float | Simulated captain availability score from 0.15 to 1.00 |
| supply_pressure_index | float | Inverse of captain availability, higher means tighter supply |
| wait_time_min | float | Estimated wait time to captain arrival |

## Pricing Features
| Column | Type | Description |
|--------|------|-------------|
| surge_multiplier | float | Dynamic surge multiplier used for private-hire products; fixed to 1.0 for Hala products |
| booking_fee_aed | float | Booking fee applied based on product and time band |
| metered_fare_aed | float | Pre-final fare produced by the pricing logic |
| final_price_aed | float | Final fare charged to the rider in AED |
| price_per_km_aed | float | Derived final fare divided by route distance |

## Outcome Features
| Column | Type | Description |
|--------|------|-------------|
| booking_status | str | Ride outcome: `Completed`, `Captain Cancelled`, `Customer Cancelled`, or `No Captain Available` |
| cancellation_reason | str | Cancellation reason, or `N/A` for completed rides |
| captain_rating | float | Synthetic captain rating, null for cancelled rides |
| customer_rating | float | Synthetic customer rating, null for cancelled rides |
| eta_deviation_min | float | Simulated deviation between promised and actual arrival |

## Notes
- The dataset intentionally mixes regulated Hala pricing and dynamic private-hire pricing in one mirror market.
- `demand_index` and `supply_pressure_index` were added to make pricing explanations more interpretable in the XAI app.
- The current generated CSV has 62 columns, which supersedes older project notes that mentioned 48 features.
