# Data Dictionary — Dubai Ride-Hailing Mirror Dataset 2025
## XPrice Project | MIT 622 Final Project | Group 1

Generated by `data/generate_dataset.py` (v3.1)

- **Records:** 165,000 synthetic rides (149,963 completed, 15,037 cancelled)
- **Columns:** 72 raw features
- **Training features:** 76 (after one-hot encoding and derived features; metadata/location-label columns are dropped and direct fare-formula columns remain excluded from model input)
- **Period:** January 1, 2025 – December 31, 2025
- **Geography:** 20 Dubai pricing zones plus Sharjah cross-border trips
- **Primary target:** `final_price_aed`

### Zone Coverage
Zones are detected from the generated pickup and dropoff coordinates using Shapely point-in-polygon matching against 134 mapped Dubai community polygons from the official GeoJSON, with nearest-centroid fallback for points outside all mapped polygons. The dataset also stores the resolved neighborhood name for each endpoint so community-level labels stay aligned with the pricing zone used by the model. The 20 zones are: Downtown, Business Bay, DIFC, Bur Dubai, Deira, Jumeirah, Al Barsha, Al Quoz, Marina, Palm Jumeirah, DXB Airport, Al Nahda, Mirdif, Silicon Oasis, Dubai Hills, JVC, Jebel Ali, Dubai South, Ras Al Khor, Sharjah.

## Trip and Route Features
| Column | Type | Description |
|--------|------|-------------|
| route_direct_distance_km | float | Haversine (great-circle) distance between pickup and dropoff coordinates |
| route_distance_km | float | Road distance from the OSRM zone-pair matrix, with +/-5% per-ride noise. Replaces the previous hardcoded lookup table. |
| route_efficiency_ratio | float | Road distance divided by direct distance; higher values indicate less direct routing |
| route_bearing_deg | float | Compass bearing from pickup to dropoff |
| salik_gates | int | Number of Salik RTA toll gates crossed, from a zone-pair table built from physical gate locations on Sheikh Zayed Road, Al Garhoud Bridge, Al Maktoum Bridge, and Al Mamzar |
| salik_cost_aed | float | Salik toll cost (salik_gates x AED 4.00). Excluded from model training - redundant with salik_gates. |
| product_type | str | Product booked: `Comfort`, `Executive`, `Hala Taxi`, `Eco Friendly`, `Electric`, `Kids`, `Hala Kids`, `Premier`, `MAX`, `Hala MAX` |
| is_hala_product | bool | `True` for regulated RTA Hala taxi products (metered fare, no surge) |
| payment_method | str | Payment method: `Credit Card`, `Cash`, `Careem Pay`, or `Careem Plus` |
| is_careem_plus | bool | `True` when payment method is `Careem Plus` (5% loyalty discount applied to private-hire fares) |

## Traffic and Timing Features
| Column | Type | Description |
|--------|------|-------------|
| traffic_index | float | Composite congestion factor on a 0.68-2.20 scale. Computed from the unified formula shared between the dataset generator and the app: base 0.82 plus 0.32 for peak, 0.10 for weekend, 0.14 times (event-1), 0.10 times (weather-1), 0.08 for airport, 0.06 times efficiency clip, plus Gaussian noise N(0, 0.04) |
| wait_time_min | float | Estimated captain arrival wait time in minutes. Excluded from model training - directly enters the Hala metered fare formula (wait x AED 0.50). |
| trip_duration_min | float | Simulated trip duration in minutes. Excluded from model training - direct component of private-hire fare formula (per_min x duration x surge). |
| avg_speed_kmh | float | Simulated average trip speed. Excluded from model training - fully determined by route_distance_km and trip_duration_min. |

---

## Demand and Supply Features
| Column | Type | Description |
|--------|------|-------------|
| demand_index | float | Composite demand score: zone_dmult x event_dmult x weather_dmult x time_mult x ramadan_mult x weekend_mult, clipped to [0.75, 3.00]. Included in model training as a legitimate demand signal. |
| captain_availability_score | float | Simulated captain supply score on [0.15, 1.00]. Excluded from model training - together with demand_index it allows exact reconstruction of surge_multiplier via: gap = demand_index minus availability; surge = round(1 + gap x 0.55, nearest 0.25). |
| supply_pressure_index | float | 1 minus captain_availability_score. Excluded from model training - redundant copy of availability. |

---

## Pricing Features
| Column | Type | Description |
|--------|------|-------------|
| surge_multiplier | float | Dynamic surge applied to private-hire products, rounded to nearest 0.25x step (1.0, 1.25, 1.5, ..., 2.5). Fixed at 1.0 for Hala products. Excluded from model training. |
| booking_fee_aed | float | Booking fee in AED, varying by product and time band (peak, off-peak, late-night). Excluded from model training - added directly to every fare. |
| metered_fare_aed | float | Pre-final fare before minimum-fare enforcement. Excluded from model features. |
| final_price_aed | float | Target variable. Final fare in AED. Includes hidden month, zone, and product policy drift plus heteroskedastic pricing noise that widens under peak, airport, event-heavy, longer, and more congested rides. Careem Plus rides receive a 5% loyalty discount on private-hire fares. |
| price_per_km_aed | float | Derived ratio: final_price_aed divided by route_distance_km. Excluded from model features. |

## Model Exclusion Summary

The following 8 columns are present in the CSV but excluded from model training to prevent data leakage:

| Excluded Column | Reason |
|----------------|--------|
| `booking_fee_aed` | Added directly to every fare |
| `trip_duration_min` | per_min x duration is a fare formula component |
| `avg_speed_kmh` | Fully determined by distance divided by duration |
| `wait_time_min` | Multiplied by 0.50 in Hala meter fares |
| `supply_pressure_index` | Exact inverse of captain_availability_score |
| `captain_availability_score` | With demand_index, reconstructs surge_multiplier exactly |
| `salik_cost_aed` | salik_gates x 4.00 - redundant with salik_gates |
| `surge_multiplier` | Direct fare formula component |
